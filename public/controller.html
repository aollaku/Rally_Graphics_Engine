Broadcast Engine Phase 9

Real FFmpeg/MediaMTX broadcast engine. Main workflow: publish video into MediaMTX, overlay the transparent RGE graphics with FFmpeg, then stream programme output to YouTube. Chromium is used only as the hidden transparent graphics renderer, not as the programme source.

Incoming Stream Input separate MediaMTX + FFmpeg containers

This does not change Output Live, Output Preview, or Controller Preview. It only feeds the Broadcast Engine overlay pipeline.

Main workflow: publish incoming video to MediaMTX (RTMP/SRT/RTSP), press “Use Incoming Stream For Overlay Outputs”, then start YouTube Main + Backup. For graphics-only streaming, start YouTube Graphics + Backup. Use primary server rtmp://a.rtmp.youtube.com/live2/YOUR_PRIMARY_KEY and backup server rtmp://b.rtmp.youtube.com/live2/YOUR_BACKUP_KEY. Keep stream keys private.
